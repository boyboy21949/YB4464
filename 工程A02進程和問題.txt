你指出的兩行確實有 實際運行風險，我解釋原因並給出 正確寫法。

1️⃣ 問題一

原程式：

Get-Process | Sort CPU -Descending | Select -First 10

問題原因：

1️⃣ Sort 在 PowerShell 不是正式名稱
正式是 Sort-Object

2️⃣ CPU 欄位有些 Process 會是 null
排序時會出錯。

正確安全版本
Get-Process |
Where-Object {$_.CPU -ne $null} |
Sort-Object CPU -Descending |
Select-Object -First 10

更清楚版本：

Get-Process |
Where-Object { $_.CPU } |
Sort-Object -Property CPU -Descending |
Select-Object -First 10 Name,CPU,Id
2️⃣ 問題二

原程式：

cd ..\

問題原因：

在 安裝器或模組內使用 cd 非常危險：

執行位置可能不同

腳本被別的腳本呼叫

PowerShell 工作目錄可能改變

企業級腳本 幾乎不使用 cd。

正確寫法

使用 腳本路徑變數：

$ScriptRoot = Split-Path -Parent $MyInvocation.MyCommand.Path

然後：

Set-Location $ScriptRoot

或直接：

$Root = Split-Path -Parent $PSScriptRoot
最安全寫法

例如 update.ps1

$Root = Split-Path -Parent $PSScriptRoot

Set-Location $Root

git pull
3️⃣ 企業級腳本標準（非常重要）

$global:Tasks = @()

function Add-InstallTask {

param(
[string]$Name,
[scriptblock]$Action
)

$task = [PSCustomObject]@{
Name = $Name
Action = $Action
}

$global:Tasks += $task

}

function Start-Installer {

Write-Host ""
Write-Host "===== AI Platform Installer ====="
Write-Host ""

foreach($task in $global:Tasks){

Write-Host "Running Task:" $task.Name

try{

& $task.Action

Write-Host "OK"

}
catch{

Write-Host "FAILED:" $_
throw

}

}

Write-Host ""
Write-Host "Installation Completed"

}

Export-ModuleMember -Function Add-InstallTask,Start-Installer
